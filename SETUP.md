# Дневник здоровья — настройка облачной версии

Один раз пройди эти шаги (~20–30 минут). Дальше приложение просто работает с телефона
и с компьютера.

---

## Шаг 1. Создать проект Supabase

1. Зайди на https://supabase.com → **Start your project** → зарегистрируйся
   (можно через Google или GitHub).
2. Нажми **New project**.
   - Name: `health-diary` (любое).
   - Database Password: придумай и **сохрани** (пригодится редко, но пусть будет).
   - Region: ближайший к тебе.
3. Жми **Create new project** и подожди ~2 минуты, пока проект поднимется.

## Шаг 2. Создать таблицу и хранилище

1. В левом меню Supabase открой **SQL Editor** → **New query**.
2. Открой файл `setup.sql` из этой папки, скопируй **весь** его текст в редактор.
3. Нажми **Run** (или Ctrl+Enter). Должно быть «Success».

Это создаёт таблицу записей, хранилище для скриншотов ВСР и правила доступа
(данные видит только их владелец).

## Шаг 3. Отключить подтверждение email

Чтобы регистрация была мгновенной, без письма со ссылкой:

1. Левое меню → **Authentication** → **Sign In / Providers** → **Email**.
2. Выключи переключатель **Confirm email** → **Save**.

(Можно и не отключать — тогда после регистрации надо будет один раз перейти по ссылке
из письма.)

## Шаг 4. Вписать ключи в config.js

1. Левое меню → **Project Settings** (шестерёнка) → **API**.
2. Скопируй два значения:
   - **Project URL** — строка вида `https://xxxxxxxx.supabase.co`
   - **Project API keys → anon public** — длинный ключ
3. Открой файл `config.js` из этой папки и вставь их между кавычками:
   ```js
   const SUPABASE_URL = 'https://xxxxxxxx.supabase.co';
   const SUPABASE_ANON_KEY = 'твой anon public ключ';
   ```
4. Сохрани файл.

> Ключ `anon public` не секретный — он рассчитан на использование в браузере.
> Доступ к данным защищён входом по паролю и правилами из `setup.sql`.

## Шаг 5. Опубликовать приложение на GitHub Pages

1. Зайди на https://github.com → зарегистрируйся (если аккаунта ещё нет).
2. Создай репозиторий: вверху справа **+** → **New repository**.
   - Repository name: `health-diary` (любое имя латиницей).
   - Выбери **Public** — для бесплатного GitHub Pages нужен публичный репозиторий.
   - Поставь галочку **Add a README file**.
   - Нажми **Create repository**.
3. Загрузи файлы приложения:
   - В репозитории нажми **Add file** → **Upload files**.
   - Открой папку `health-diary-cloud` на компьютере и перетащи в окно браузера
     **все 6 файлов**: `index.html`, `styles.css`, `app.js`, `config.js`,
     `setup.sql`, `SETUP.md`.
   - Важно: перетаскивай сами файлы, **не** папку целиком и **не** другие папки
     проекта (особенно не `health-data.json` — это твои личные данные, в публичный
     репозиторий их класть нельзя).
   - Внизу нажми **Commit changes**.
4. Включи GitHub Pages:
   - В репозитории открой **Settings** → в левом меню **Pages**.
   - **Build and deployment** → Source: **Deploy from a branch**.
   - Branch: **main**, папка: **/ (root)** → **Save**.
5. Подожди ~1 минуту и обнови страницу Pages — вверху появится ссылка вида
   `https://твой-логин.github.io/health-diary/`. Это адрес приложения.

> Публичным будет только код приложения. Ключ `anon public` в нём — не секретный
> (см. Шаг 4). Личные записи и скриншоты остаются в Supabase под защитой пароля.

> Если позже поменяешь файлы (например, впишешь ключи в `config.js`) — в репозитории
> снова **Add file** → **Upload files**, перетащи изменённые файлы, **Commit changes**.
> Сайт обновится через минуту.

## Шаг 6. Открыть на телефоне

1. Открой ссылку приложения в браузере телефона.
2. Введи email и пароль → нажми **Регистрация**. Аккаунт создан, дневник открылся.
3. Добавь на главный экран:
   - **iPhone (Safari):** кнопка «Поделиться» → «На экран Домой».
   - **Android (Chrome):** меню ⋮ → «Добавить на главный экран».
4. На компьютере открой ту же ссылку и войди тем же email/паролем — данные общие.

---

## Как идёт вечерний разбор

Данные лежат в облаке, поэтому ПК для записи не нужен. Для анализа дня:

1. На компьютере открой приложение, нажми **«Экспорт JSON»** — скачается файл
   `health-data.json` (обычно в папку «Загрузки»).
2. Скажи Claude: «проанализируй день» — он прочитает скачанный файл.
3. Скриншоты ВСР хранятся в Supabase; в экспорт попадают временные ссылки на них.
   Если нужен детальный разбор картинки — можно скинуть скриншот прямо в чат.

---

## Если что-то не работает

- **«Не заполнен config.js»** — проверь Шаг 4, значения должны быть без слова `ВСТАВЬ`.
- **Не входит / «Неверный email или пароль»** — если только что регистрировался и не
  отключал подтверждение email (Шаг 3), перейди по ссылке из письма.
- **Запись не сохраняется** — убедись, что `setup.sql` отработал без ошибок (Шаг 2).
- **Проект Supabase «на паузе»** — бесплатный тариф засыпает после ~недели простоя;
  просто открой проект в панели Supabase, он проснётся.
